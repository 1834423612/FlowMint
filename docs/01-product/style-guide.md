# Style Guide

## Design tokens

- Primary: indigo / blue family
- Success: green family
- Warning: amber family
- Danger: red family
- Background: dark neutral
- Card: slightly raised dark panel
- Border radius: medium to large, not overly round

## Typography

- Sans UI font for product interface
- Monospace only for code, IDs, or JSON blocks

## Spacing

- Prefer generous spacing over dense enterprise clutter
- Use repeatable spacing scales
- Forms and config panels should remain visually aligned

## Icons

- Use **Iconify** as the primary icon system
- Avoid mixing unrelated icon libraries as the default design language

## Interaction patterns

- clear hover
- visible selected state
- keyboard-friendly forms where practical
- destructive actions require confirmation
